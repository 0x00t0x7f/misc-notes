# hidevops
是一种 devops 数据和指标展示的看板平台，汇集了软件生命周期以及项目管理过程中产生的数据、各项指标。 帮助各个部门及时发现和度量软件开发过程中的各种问题，以优化和改进开发流程，规范开发行为。
可信过程服务）是面向业务层（Django）与数据层（ORM）的全链路可信性平台，负责 代码清洗、可信度量、可信测试、漏洞追踪、端到端追溯 等核心业务的数据采集、加工、展示与导出。系统通过统一的公共工具库（缓存、公式计算、权限校验、通用数据处理）为各业务子模块提供底层支撑，帮助组织实现软件过程的可视化、可审计与可度量。

主体采用 django 3.2 + redis + mysql 框架，python（3.10.2）版本
四个代码仓：
+ 前端仓：vue，本地以服务形式运行、线上是编译打包后的静态文件
+ 后端服务仓：django驱动
+ 后端定时任务仓：调用外部接口同步数据的服务，以celery服务运行
+ 后端数据接口服务仓（建议改为fastapi）: 对外提供数据接口服务，包含外部向hidevops上报数据接口、从hidevops查询数据。

**环境**  
+ alpha为开发环境（开发自测、上线风险高的代码必须在alpha环境自测）
+ lambda、gamma 为测试环境，作为结项时进行功能测试、压力测试环境
+ beta 为生产环境的异地容灾环境，prod是生产环境

其中 alpha、beta环境部署不需要 流程号；beta和prod连接的生产数据库，有UEM埋码，alpha、gamma、lambda连接的测试库

**核心交互**  
+ 上游：代码仓库、CI/CD系统、需求管理平台等向本服务推送MR、需求、缺陷等原始数据。
+ 本系统：解析、清洗、关联并持久化至MYSQL，通过redis实现热点数据缓存，提供restful api供前端（vue）或第三方服务系统调用
+ 下游：BI报表系统、质量审计平台、自动化测试框架通过API拉取可信度量、漏洞追踪、E2E追溯等结果进行进一步分析或展示。

**权限**  
多个部门租户之间数据、权限隔离，每个部门租户权限根据模块分类，每个分类均包含本模块的管理权限，有此权限的用户才能编辑本模块的权限。但除管理员用户、群组外不能赋予其他用户模块的管理权限。

#### 漏洞管理（重点）
**漏洞度量**
+ 全量管理率

**单漏洞多版本**
+ 一个漏洞——多个受影响版本（漏洞编号、受影响版本数、应修完成率、类别、产品/软件名称、漏洞来源、级别、CVSS得分、责任人、责任部门、创建时间、根因分析）

**单版本多漏洞**
+ 版本（生命周期、修补中漏洞数、漏洞总数）

**自研漏洞登记入库**
+ 部门名称（部门漏洞SLO owner、部门CSRE、感知中漏洞SLO超期数、感知中漏洞总数、..）

**漏洞评估（表头：版本漏洞感知）**
+ 部门维度（部门名称、部门漏洞SLO owner、部门CSRE、SLO到期、SLO达成、SQL未达成、SLO达成率..）
+ 版本维度

**漏洞修补（表头：修补中）**
+ 部门维度（部门名称、部门漏洞SLO owner、部门CSRE、SLO超期数、SQL未超期数、SQL即将超期数..）
+ 版本维度

其中点击进去某个漏洞：
+ 漏洞编号
+ 发现人
+ 来源
+ 社区/供应商计划纰漏时间
+ 攻击矢量：CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H  AV:L 表示 Attack Vector(AV) L表示local，N表示network   其中类似
