# 需求

**优惠码生成**  
+ 后台管理系统生成优惠码（单次、批量，PS: 只有超管可以创建、用户只能使用）
+ 优惠码可配置过期时间，配置渠道、配置使用次数（默认1次）、可选：绑定某用户、适用商品范围
+ 优惠规则集配置（规则编号、disacount_type、disacount_type、规则描述）如 1、percentage、0.75、限时75折购买； 2、fixed、5、5元优惠券
+ 优惠码模型
  - ID
  - CODE
  - 渠道（闲鱼、抖音、淘宝）
  - 范围（适用范围）
  - 最大使用次数（如果最大使用次数>1，则无法绑定具体某一个用户）
  - 已使用次数
  - 绑定用户（如果绑定某个用户，则只能使用1次）
  - 锁定/解锁（用于分发给用户时锁定该优惠码）
  - 是否有效（is_active）
  - 有效期（永久、一周、一个月、三个月、时间范围：自定义时间范围；默认永久）

**优惠码使用**  
+ 用户登录后，在购买会员页添加优惠码输入框/在付款弹窗中添加（UI交互待定）

**优惠码校验**  
+ 需登录
+ 优惠码接口限流防刷（slowapi/redis）
+ 判断是否存在、优惠码特征
+ 判断是否过期
+ 判断是否有效
+ 判断是否专属
+ 判断是否已被使用
+ 判断适用范围是否一致
+ 付款时执行优惠扣减（发送通知事件/kfk生产消费模型、最终一致性）

**使用后处理**  
+ 使用记录（入库）
  - id
  - 优惠码ID
  - 用户ID
  - 订单ID
  - 使用时间
+ 记录日志
