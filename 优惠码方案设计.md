# 需求

**优惠码生成**  
+ 后台管理系统超管页面左侧栏新增一个生成优惠码页签（单次、批量，PS: 只有超管角色可以创建）
+ 优惠码可配置过期时间，配置渠道、配置使用次数（默认1次）、可选：绑定某用户、适用商品范围
+ 优惠规则集配置（规则编号、disacount_type、value、规则描述）如 1、percentage、0.75、限时75折购买； 2、fixed、5、5元优惠券
+ 操作：删除（已使用的优惠码禁止删除）
+ 优惠码模型
  - ID
  - CODE（唯一）
  - 渠道（闲鱼、抖音、淘宝）
  - 范围（适用范围）
  - 最大使用次数（如果最大使用次数>1，则无法绑定具体某一个用户）
  - 已使用次数
  - 绑定用户（如果绑定某个用户，则只能使用1次；默认为空不绑定任何用户）
  - 是否有效（is_active、切换按钮可以切换）
  - 有效期（永久、一天、三天、一周、一个月、三个月;默认永久）
  - 创建时间
  - 到期时间（如果有效期是永久，则到期时间为空）
  - 更新时间

**优惠码使用**  
+ 用户登录后，在购买会员页添加优惠码输入框/在付款弹窗中添加（UI交互待定）

**优惠码校验**  
+ 需登录
+ 优惠码接口限流防刷（slowapi/redis）
+ 判断是否存在、优惠码特征
+ 判断是否过期
+ 判断是否有效
+ 判断是否专属
+ 判断是否已被使用
+ 判断适用范围是否一致
+ 付款时执行优惠扣减（发送通知事件/kfk生产消费模型、最终一致性）

**使用后处理**  
+ 使用记录模型（入库）
  - id
  - 优惠码ID
  - 用户ID
  - 订单ID
  - 使用时间
+ 记录日志  logger.info
